Embedding Fuse in your website is simple. There are two steps:
1. Add the [fuse-importer NPM package](#adding-the-npm-package) to your application.
2. Add a small [JavaScript snippet](#launching-an-importer) to open your Fuse importer.

## Adding the NPM Package
<Tabs>
  <Tab title="YARN">
   To add Fuse, do:

    ```bash
    yarn add fuse-importer
    ```

    To import it for use:

    ```typescript
    import FuseImporter from "fuse-importer";
    ```

  </Tab>
  <Tab title="NPM">
   To add Fuse, do:

    ```bash
    npm install fuse-importer --save
    ```

    To import it for use:

    ```typescript
    import FuseImporter from "fuse-importer";
    ```

  </Tab>
  <Tab title="CDN">
    Fuse is also hosted on the [unpkg.com](https://unpkg.com/) CDN. Make sure to place the script tag before you initialize the importer.

    ```html
    <script type="text/javascript" src="https://unpkg.com/fuse-importer@latest"></script>
    ```

  </Tab>
</Tabs>

## Launching an Importer

To display the Fuse Importer, you need to use the `getSessionToken` method for secure authentication. This involves setting up your backend to handle session token requests and integrating it with your frontend.

#### General Steps:

1. **Backend Setup**: Create an endpoint that calls the Fuse API to obtain a session token. This endpoint should use your `apiToken` for authentication and include the `templateSlug` and optional metadata.

2. **Frontend Setup**: Define the `getSessionToken` method to fetch the session token from your backend. Use this token to initialize and display the importer.

#### Example for Node.js

##### Backend (Node.js with Express):
```javascript
const express = require('express');
const axios = require('axios');
const app = express();
app.use(express.json());

app.post('/api/getSessionToken', async (req, res) => {
  const { templateSlug, metadata } = req.body;
  try {
    const response = await axios.post('https://fuse.flatirons.com/api/v1/importer/sessions', {
      templateSlug,
      metadata
    }, {
      headers: { 'Authorization': `Bearer YOUR_API_TOKEN` }
    });
    res.json({ sessionToken: response.data.sessionToken });
  } catch (error) {
    res.status(500).json({ error: 'Failed to get session token' });
  }
});

app.listen(3000, () => console.log('Server running on port 3000'));
```

For more information on the `/api/v1/importer/sessions` endpoint, refer to the [Import Session API documentation](/apis/resources/import-session).

For information on obtaining the `apiToken`, see the [Authentication documentation](/apis/authentication).

##### Frontend (React):
```javascript
import FuseImporter from 'fuse-importer';
import React from 'react';

const App = () => {
  const importer = new FuseImporter();

  importer.getSessionToken = async () => {
    const response = await fetch('/api/getSessionToken', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ templateSlug: 'YOUR_TEMPLATE_SLUG', metadata: {} })
    });
    const data = await response.json();
    return data.sessionToken;
  };

  const showImporter = () => {
    importer.show();
  };

  return (
    <button onClick={showImporter}>Show Importer</button>
  );
};

export default App;
```

**Note**: Metadata should be in JSON format and will be associated with the importer session. This can be useful for tracking events or user activity.

#### Without a Backend

If you don't have a backend, you can use the `getFuseClientSideToken` function provided by the `fuse-importer` package:

```javascript
import FuseImporter, { getFuseClientSideToken } from "fuse-importer";

const importer = new FuseImporter();

importer.getSessionToken = getFuseClientSideToken({
  importer_id: "YOUR_IMPORTER_ID",
  api_key: "YOUR_API_KEY"
});

importer.show();
```

Replace `YOUR_IMPORTER_ID` and `YOUR_API_KEY` with your actual importer ID and API key.

**Note**: While this method is simpler, it's less secure as it exposes your API key in the client-side code. Use this approach only for testing or in trusted environments. For production use, we recommend implementing a backend solution as described in the previous section.
